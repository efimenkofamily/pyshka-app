# Архитектура проекта «Клуб Пышка»
## Техническая документация, структура БД и подсистем (Релизная версия 4.0 — React Frontend)

Этот документ содержит актуальное описание архитектуры приложения, структуры таблиц в базе данных Supabase, логики парсера, конфигураций маржинальности, десктопной веб-админки и нового клиентского приложения на React.

---

## 📂 1. Структура репозитория и файлы проекта

* `/client` — **(Новое)** Клиентское WebApp-приложение для Telegram-бота (SPA на React + Vite). Включает в себя каталог, умную корзину, профиль пользователя и систему оформления заказов.
* `/admin` — Десктопное SPA-приложение на React + Vite для разработчиков и администраторов. Развернуто в подпапке текущего репозитория и деплоится на платформу Netlify с изолированным указанием базовой директории сборки (`admin`).
* `Google Apps Script` — Внешний парсер прайс-листов из Google Таблиц в Supabase, управляемый по HEX-цветам ячеек.

## 📱 3. Клиентское WebApp приложение (`/client`)

Клиентская часть полностью переписана на **React** для обеспечения высокой производительности, модульности и современного UX/UI дизайна. Приложение интегрировано с Telegram WebApp API.

### Основные компоненты и модули:
* **Основной контроллер (`App.jsx`):** Управляет авторизацией (RLS проверка прав `approved`), навигацией между экранами (Каталог, Оформление, Профиль) и глобальными состояниями (Корзина, Фильтры).
* **Умный Каталог:** * Автоматически группирует сырой список товаров из базы по ключу `[Бренд + Линейка]`.
    * Связывает сгруппированные карточки с таблицей `series_images` для вывода обложек.
    * Поддерживает два вида отображения: Сетка (карточки с фото) и Список (компактный вид).
    * Сквозная фильтрация: поиск по тексту, двойные Range-ползунки по цене и крепости, сортировка (по алфавиту/цене).
* **Оформление заказа (`Checkout.jsx`):** Модуль подсчета итоговой суммы (с учетом порога бесплатной доставки из `config`), выбора типа получения (курьер/самовывоз) и отправки валидного JSON-слепка `items` в базу.
* **Профиль пользователя (`Profile.jsx`):** * **Трекер заказа:** Динамически ищет активный заказ (status_id 1-4) и выводит красивый прогресс-бар с раскрывающимся списком товаров. Данные статусов (label) тянутся напрямую из таблицы `order_statuses`.
    * **История:** Вложенный под-экран с полным списком прошлых заказов пользователя (RLS защита — пользователь видит только свои записи).

## 💻 4. Функциональные модули Веб-Админки (`/admin`)

* **Авторизация (`Login.jsx`):** Интеграция с Telegram Login Widget. Доступ открывается исключительно для статусов `developer` и `admin`. Сессия кэшируется в `localStorage`.
* **Глобальный каркас (`Layout.jsx`):** Десктопная сетка со сквозным Sidebar слева для навигации.
* **Управление пользователями (`Users.jsx`):** Таблица со списком клиентов. Позволяет менять уровни доступа (`status`) через динамически окрашенные селекты и переходить в чат с пользователем.
* **Диспетчер заказов (`Orders.jsx`):** Интерактивный журнал продаж с системой раскрывающихся строк для просмотра состава заказа (`items JSON`).
    * **Финансовый дашборд:** Общая выручка, Суммарная себестоимость закупки, Общая чистая прибыль со всех заказов.
    * **Калькулятор себестоимости и маржи:** На основе вложенных JSON объектов рассчитывает затраты на закупку и выводит чистую маржу индивидуально по каждому заказу.
* **Каталог товаров и маржа (`Products.jsx`):** Финансовый мониторинг номенклатуры. Отображает закупку, динамически рассчитывает розницу на базе таблицы `config` и показывает плановую прибыль с единицы товара.
* **Конфигуратор магазина (`Settings.jsx`):** Управление расписанием дропов, лимитами курьера, базовыми наценками и JSON-коэффициентами по категориям товаров.
