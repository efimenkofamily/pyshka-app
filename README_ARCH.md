# Архитектура проекта «Клуб Пышка»
## Техническая документация, структура БД и подсистем (Релизная версия 4.5 — Fullstack App)

Этот документ содержит актуальное описание архитектуры приложения, структуры таблиц в базе данных Supabase, логики парсера, конфигураций маржинальности, десктопной веб-админки, клиентского SPA-приложения на React и подсистемы маршрутизации алертов.

---

## 📂 1. Структура репозитория и файлы проекта

* `/client` — Клиентское WebApp-приложение для Telegram-бота (SPA на React + Vite). Включает в себя каталог, интерактивную корзину, профиль пользователя с плавными анимациями, систему оформления заказов, мобильную админку и стартовую анкету. Деплоится на Netlify (директория `dist`).
* `/admin` — Десктопное SPA-приложение на React + Vite для разработчиков и администраторов. Содержит продвинутый журнал заказов с функционалом ручной модификации чеков, калькуляторы прибыли, страницу изолированного управления уведомлениями и маржи. Деплоится на Netlify (базовая директория `admin`).
* `Google Apps Script` — Внешний бэкенд-парсер прайс-листов из Google Таблиц в базу данных Supabase.
* `Supabase Edge Functions` — Бессерверный реактивный слой бэкенда (`notify-status`), обрабатывающий системные хуки таблиц базы данных и осуществляющий маршрутизацию транзакционных пушей.

## ⚙️ 2. Логика работы парсера (Google Apps Script)

Парсер является связующим звеном между прайсами поставщика и нашей витриной. Он работает по следующим принципам:
* **Цветовая маршрутизация (HEX):** Скрипт читает цвета ячеек в Google Таблицах. По цвету определяется иерархия: маркер Производителя (`#434343`), маркер Линейки (`#fff2cc`). Черные строки (например, ice-версии) аппаратно игнорируются (`#000000`).
* **Умный парсинг расходников (RegEx Расщепление):** В таблицах поставщика расходники (картриджи и испарители) часто записаны в одну строку с перечислением сопротивлений (например: *«Vaporesso LUXE Q | 0.6om, 0.8om (2ml)»*). Парсер автоматически "расщепляет" такие строки на самостоятельные позиции в БД:
    * Название модели устройства (*Vaporesso LUXE Q*) записывается в колонку `series` (Линейка).
    * Сопротивления (*0.6om*) и объем бака (*2ml*) извлекаются через регулярные выражения и записываются в колонку `flavor` (Вкус).
    * **Результат:** В клиентском каталоге устройства отображаются как отдельные карточки линеек, а доступные сопротивления — как раскрывающийся список внутри этой карточки.

## 🎛 3. Матрица и Маршрутизация Уведомлений (Telegram Алерты)

Система алертов полностью отвязана от монолитного кода приложений и работает реактивно на базе триггерных вебхуков Supabase и бессерверной функции Edge Function `notify-status` (`index.ts`).

### Логика работы подсистемы:
1. **База данных (Database Webhooks):** При совершении операций `INSERT` или `UPDATE` в таблицах `users`, `orders` или `config`, СУБД генерирует payload и отправляет его по протоколу POST в Edge Function.
2. **Сканирование конфигурации:** Функция первым делом запрашивает поле `notification_matrix` из таблицы `config` (строка `id = 1`) и кэширует правила доступа в памяти.
3. **Фильтрация по ролям (Вахтер):** Для каждого получателя вычисляется его текущая роль (`developer`, `admin`, `approved`, `pending`). Если матрица содержит `false` для связки `[Системное событие][Роль]` — отправка сообщения блокируется. Это позволяет изолированно отключать уведомления для клиентов во время проведения технических тестов.

### Список системных событий матрицы:
* `NEW_USER_PENDING` — Оповещение администрации о поступлении новой заявки на вступление в клуб.
* `NEW_ORDER` — Отправка структурированного чека и подтверждения клиенту в ЛС.
* `NEW_ORDER_STAFF` — Мгновенное дублирование состава и метаданных нового заказа (включая комментарии и адреса) в ЛС администрации и разработчикам.
* `ORDER_STATUS_CHANGED` — Информирование покупателя об изменении статуса его заказа менеджером.
* `STORE_OPENED` — Автоматический массовый броадкаст-пуш по всей базе пользователей об открытии магазина (дропе) при изменении `open_date`.
* `STORE_CLOSED` — Массовый броадкаст-пуш о закрытии магазина при обновлении `close_date`.

## 💻 5. Функциональные модули Десктопной Админки (`/admin`)

* **Авторизация (`Login.jsx`):** Интеграция с Telegram Login Widget. Доступ открывается исключительно для статусов `developer` и `admin`.
* **Диспетчер заказов (`Orders.jsx`):** Интерактивный журнал продаж со сквозной аналитикой и расширенным контекстным управлением.
    * **Контекстное меню (Три точки):** Кнопка вызова контекстного меню для удаления заказа или запуска режима глубокого редактирования чека.
    * **Двухколоночный пульт модификации:** Инструмент для ручного урегулирования пересортицы. Содержит состав корзины покупателя и реактивную систему поиска замен по актуальной номенклатуре склада.
    * **Финансовый процессор:** При редактировании чека «на лету» санитизирует закупочные цены, применяет формулы маржинальности из `config`, учитывает пороги доставки и рассчитывает новую сумму.
* **Каталог товаров и маржа (`Products.jsx`):** Модуль финансового мониторинга номенклатуры. Динамически рассчитывает розницу на базе `config` и показывает плановую прибыль с единицы товара.
* **Управление Уведомлениями (`Notifications.jsx`):** Изолированная страница-пульт управления алертами. Реализует глубокое JSON-копирование состояния во избежание мутаций стейта React. Сохраняет настройки матрицы в `config` атомарно, исключая перезапись финансовых полей.

## 📱 6. Клиентское WebApp приложение (`/client`)

### Основные компоненты и модули:
* **Основной контроллер (`App.jsx`):** Управляет авторизацией, навигацией и состояниями. Блокирует каталог для обычных покупателей вне расписания дропов.
* **Умный Каталог:**
    * **Улучшенный UX/UI:** Иерархия карточек смещена в сторону линеек (Линейка выделяется крупно сверху, Производитель ниже). Технические префиксы (например, `new`) удаляются на фронтенде.
    * **Светофорные бейджи крепости:** Зеленый фон для 0-20 мг, оранжевый для 21-40 мг, красный для 41+ мг.
    * **Плавные Анимации:** Раскрытие списков вкусов переведено на производительную CSS Grid анимацию (`grid-template-rows: 0fr -> 1fr`).
* **Оформление заказа (`Checkout.jsx`):**
    * **Интерактивная корзина:** Инкремент и декремент количества товаров (`+` / `-`) непосредственно в корзине (блокировка ниже 1 шт).
    * **Swipe-to-Delete:** Нативный паттерн удаления позиций — по свайпу справа налево выезжает кнопка удаления.
* **Профиль пользователя (`Profile.jsx`):**
    * **Дизайн чека:** Двухстрочный паттерн отображения товаров с разделительными линиями. Правый блок с ценой аппаратно защищен от переносов (`flexShrink: 0`).
    * **Grid Анимация:** Плавное раскрытие состава активного заказа.
    * **Кнопка изменения:** Позволяет отменить заказ в БД, вернуть товары в корзину и перейти в каталог для редактирования (до статуса `edit_lock_status`).
