# Архитектура проекта «Клуб Пышка»
## Техническая документация, структура БД и подсистем (Релизная версия 3.2)

Этот документ содержит актуальное описание архитектуры приложения, структуры таблиц в базе данных Supabase, логики парсера, конфигураций маржинальности и полностью перенесенных функций десктопной веб-админки.

---

## 📂 1. Структура репозитория и файлы проекта

* `index.html` / `catalog.html` / `catalog.css` / `catalog.js` — Клиентское WebApp-приложение для Telegram-бота (каталог, корзина, оформление заказов).
* `Google Apps Script` — Внешний парсер прайс-листов из Google Таблиц в Supabase, управляемый по HEX-цветам ячеек.
* `/admin` — Десктопное SPA-приложение на React + Vite для разработчиков и администраторов. Развернуто в подпапке текущего репозитория и деплоится на платформу Netlify с изолированным указанием базовой директории сборки (`admin`).

| **id** | BigInt | Telegram ID пользователя (Primary Key) |

| **delivery_type** | String | `pickup` (самовывоз) или `delivery` (курьер) |
| **address** | String | Адрес доставки |
| **status_id** | Integer | Связь с таблицей `order_statuses` |

| Колонка | Тип данных | Описание |

## 💻 3. Функциональные модули Веб-Админки (`/admin`)

* **Авторизация (`Login.jsx`):** Интеграция с Telegram Login Widget. Доступ открывается исключительно для статусов `developer` и `admin`. Сессия кэшируется в `localStorage`.
* **Глобальный каркас (`Layout.jsx`):** Десктопная сетка со сквозным Sidebar слева для навигации. Интегрирована библиотека векторных иконок `@ant-design/icons`.
* **Управление пользователями (`Users.jsx`):** Таблица со списком клиентов. Позволяет менять уровни доступа (`status`) через динамически окрашенные селекты, переходить в чат с пользователем в один клик и удалять аккаунты.
* **Диспетчер заказов (`Orders.jsx`):** Интерактивный журнал продаж с системой раскрывающихся строк (Accordion-style) для бесшовного просмотра состава заказа (`items JSON`) прямо в таблице.
    * **Финансовый дашборд:** Наверху страницы интегрирован блок сквозной аналитики (Общая выручка, Суммарная себестоимость закупки, Общая чистая прибыль со всех заказов).
    * **Калькулятор себестоимости и маржи:** На основе вложенных в JSON объектов система на лету извлекает данные `price_10` и `qty`, рассчитывает затраты на закупку для каждой позиции и выводит чистую маржу продажи (`total_price - себестоимость`) индивидуально по каждому заказу.
* **Каталог товаров и маржа (`Products.jsx`):** Модуль финансового мониторинга номенклатуры. Отображает закупку, динамически рассчитывает розницу на базе коэффициентов из таблицы `config` и показывает плановую прибыль с единицы товара.
* **Конфигуратор магазина (`Settings.jsx`):** Двухколоночная адаптивная сетка карточек. Позволяет управлять расписанием дропов (с ISO-конвертацией часовых поясов для календарей), лимитами курьера, базовыми наценками и JSON-коэффициентами по категориям товаров.
