# Архитектура проекта «Клуб Пышка»
## Техническая документация, структура БД и подсистем (Релизная версия 4.2 — Fullstack App)

Этот документ содержит актуальное описание архитектуры приложения, структуры таблиц в базе данных Supabase, логики парсера, конфигураций маржинальности, десктопной веб-админки и клиентского SPA-приложения на React.

---

## 📂 1. Структура репозитория и файлы проекта

* `/client` — Клиентское WebApp-приложение для Telegram-бота (SPA на React + Vite). Включает в себя каталог, умную корзину, профиль пользователя с плавными анимациями, систему оформления заказов, мобильную админку и стартовую анкету. Деплоится на Netlify (директория `dist`).
* `/admin` — Десктопное SPA-приложение на React + Vite для разработчиков и администраторов. Содержит продвинутый журнал заказов с функционалом ручной модификации чеков, калькуляторы прибыли и маржи. Деплоится на Netlify (базовая директория `admin`).
* `Google Apps Script` — Внешний парсер прайс-листов из Google Таблиц в Supabase, управляемый по HEX-цветам ячеек.

## 💻 3. Функциональные модули Десктопной Админки (`/admin`)

* **Авторизация (`Login.jsx`):** Интеграция с Telegram Login Widget. Доступ открывается исключительно для статусов `developer` и `admin`.
* **Диспетчер заказов (`Orders.jsx`):** Интерактивный журнал продаж со сквозной аналитикой и расширенным контекстным управлением.
    * **Контекстное меню (Три точки):** В правую часть каждой строки заказа интегрирована кнопка вызова контекстного меню (`MoreOutlined`), позволяющая оперативно удалить заказ из БД или запустить режим глубокого редактирования чека.
    * **Двухколоночный пульт модификации (Модальное окно):** Инструмент для ручного урегулирования пересортицы или отсутствия товаров у поставщика. 
        * *Левая колонка:* Отображает текущий состав корзины покупателя с кнопками инкремента/декремента (`+`/`-`) и удаления позиций.
        * *Правая колонка:* Содержит реактивную систему сквозного поиска по всей актуальной номенклатуре склада (`availability = 'есть'`) для быстрого подбора замен.
        * *Финансовый процессор:* При добавлении или удалении позиций модальное окно «на лету» санитизирует закупочные цены, применяет формулы маржинальности из `config`, учитывает пороги бесплатной доставки и рассчитывает новую `total_price` перед записью в базу.
* **Каталог товаров и маржа (`Products.jsx`):** Модуль финансового мониторинга номенклатуры. Динамически рассчитывает розницу на базе таблицы `config` и показывает плановую прибыль с единицы товара.

## 📱 4. Клиентское WebApp приложение (`/client`)

### Основные компоненты и модули:
* **Основной контроллер (`App.jsx`):** Управляет авторизацией, навигацией и глобальными состояниями. Автоматически перехватывает отсутствие пользователя в базе данных и отправляет его на экран анкеты (`Registration.jsx`).
    * **Контроль Дропов (`isShopOpen`):** На верхнем уровне блокирует каталог для обычных покупателей вне установленного расписания, подменяя его страницей-заглушкой «Наводим порядок».
* **Профиль пользователя (`Profile.jsx`):** * **Улучшенный дизайн состава чека:** В блоках текущего заказа и истории продаж внедрен двухстрочный паттерн отображения товаров. Название линейки (или бренда) выносится на первую строку и выделяется насыщенным жирным шрифтом (`fontHeight: '800'`), а вкус располагается на второй строке регулярным приглушенным цветом. Позиции визуально отделены друг от друга тонкими разделительными линиями. Правый блок (количество и стоимость) аппаратно защищен от переносов (`flexShrink: 0`).
    * **Плавный выезд (Grid Анимация):** Процесс раскрытия и сворачивания состава активного заказа переведен с жесткого монтирования на современную CSS Grid анимацию (`gridTemplateRows: isOrderExpanded ? '1fr' : '0fr'`). Это позволило добиться эффекта плавного скольжения контента без резких прыжков интерфейса.
    * **Кнопка изменения:** Позволяет покупателю в один клик отменить свой текущий заказ в базе данных, распаковать массив товаров `items` обратно в локальную корзину и бесшовно вернуться в каталог для редактирования (доступно строго до достижения статуса `edit_lock_status`).
