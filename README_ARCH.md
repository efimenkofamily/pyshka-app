# Архитектура проекта «Клуб Пышка»
## Техническая документация, структура БД и подсистем (Релизная версия 4.3 — Fullstack App)

Этот документ содержит актуальное описание архитектуры приложения, структуры таблиц в базе данных Supabase, логики парсера, конфигураций маржинальности, десктопной веб-админки и клиентского SPA-приложения на React.

---

## 📂 1. Структура репозитория и файлы проекта

* `/client` — Клиентское WebApp-приложение для Telegram-бота (SPA на React + Vite). Включает в себя каталог, интерактивную корзину, профиль пользователя с плавными анимациями, систему оформления заказов, мобильную админку и стартовую анкету. Деплоится на Netlify (директория `dist`).
* `/admin` — Десктопное SPA-приложение на React + Vite для разработчиков и администраторов. Содержит продвинутый журнал заказов с функционалом ручной модификации чеков, калькуляторы прибыли и маржи. Деплоится на Netlify (базовая директория `admin`).
* `Google Apps Script` — Внешний парсер прайс-листов из Google Таблиц в Supabase, управляемый по HEX-цветам ячеек.

## 💻 3. Функциональные модули Десктопной Админки (`/admin`)

* **Авторизация (`Login.jsx`):** Интеграция с Telegram Login Widget. Доступ открывается исключительно для статусов `developer` и `admin`.
* **Диспетчер заказов (`Orders.jsx`):** Интерактивный журнал продаж со сквозной аналитикой и расширенным контекстным управлением.
    * **Контекстное меню (Три точки):** В правую часть каждой строки заказа интегрирована кнопка вызова контекстного меню, позволяющая оперативно удалить заказ из БД или запустить режим глубокого редактирования чека.
    * **Двухколоночный пульт модификации (Модальное окно):** Инструмент для ручного урегулирования пересортицы или отсутствия товаров у поставщика. 
        * *Левая колонка:* Отображает текущий состав корзины покупателя с кнопками инкремента/декремента (`+`/`-`) и удаления позиций.
        * *Правая колонка:* Содержит реактивную систему сквозного поиска по всей актуальной номенклатуре склада для быстрого подбора замен.
        * *Финансовый процессор:* При добавлении или удалении позиций модальное окно «на лету» санитизирует закупочные цены, применяет формулы маржинальности из `config`, учитывает пороги бесплатной доставки и рассчитывает новую `total_price` перед записью в базу.
* **Каталог товаров и маржа (`Products.jsx`):** Модуль финансового мониторинга номенклатуры. Динамически рассчитывает розницу на базе таблицы `config` и показывает плановую прибыль с единицы товара.

## 📱 4. Клиентское WebApp приложение (`/client`)

### Основные компоненты и модули:
* **Основной контроллер (`App.jsx`):** Управляет авторизацией, навигацией и глобальными состояниями.
    * **Контроль Дропов (`isShopOpen`):** Блокирует каталог для обычных покупателей вне установленного расписания, подменяя его страницей-заглушкой «Наводим порядок».
* **Умный Каталог:** * **Улучшенный UX/UI:** Иерархия карточек смещена в сторону продуктовых линеек (Линейка выделяется крупным жирным шрифтом сверху, Производитель отображается ниже). Технические префиксы в базе (например, `new`) автоматически удаляются на фронтенде через регулярные выражения.
    * **Динамические бейджи крепости:** Внедрена светофорная цветовая индикация крепости (зеленый фон для 0-20 мг, оранжевый для 21-40 мг, красный для 41+ мг).
    * **Плавные анимации:** Процесс раскрытия списков вкусов в режимах сетки и списка переведен на производительную CSS Grid анимацию (`grid-template-rows: 0fr -> 1fr`), исключающую резкие скачки интерфейса.
* **Оформление заказа (`Checkout.jsx`):**
    * **Интерактивная корзина:** Добавлена возможность инкремента и декремента количества товаров (`+` / `-`) непосредственно в корзине (с аппаратной блокировкой снижения ниже 1 шт).
    * **Swipe-to-Delete:** Реализован нативный мобильный паттерн удаления позиций — по свайпу справа налево плавно выезжает кнопка удаления товара.
* **Профиль пользователя (`Profile.jsx`):** * **Дизайн состава чека:** Двухстрочный паттерн отображения товаров с тонкими разделительными линиями. Название линейки выделяется `fontHeight: '800'`. Правый блок (количество и стоимость) аппаратно защищен от переносов (`flexShrink: 0`).
    * **Кнопка изменения заказа:** Позволяет покупателю отменить свой текущий заказ в базе данных, вернуть массив товаров `items` в корзину и бесшовно перейти в каталог для редактирования (до статуса `edit_lock_status`).
    * **Grid Анимация:** Плавное раскрытие состава активного заказа, аналогично каталогу.
